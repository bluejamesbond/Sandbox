Universe: 2D Game Engine

The Humble Roots

The Universe 2D Game Engine is inpired by the Lisp language Racket which I had learned during a summer spent at Brown University with 2 computer science professorts (Shriram Krinamurthi, Phd. and Casey Marks, Phd.) The former had helped write the language Racket to ease Java users into Lisp langauges which are remarkable different in their approach when it comes to user-code interaction.

The Purpose of Such A Software

This application allows users to have an environment controlled by an engine where they can experiment with mouse listeners, image manipulations, media player, links, etc. This way, beginners have a generalized prespective of the language while merging advance code. Allows the user to also pause the game/engine or just pause the game itself.

Java >= Visual C++ (UI Battle)

This application proves that the same level of application graphics (user interface level) can be applied through Java custom painting as Microsoft's Zune media player, an endeavor completed through Visual C++. Goes to say, Java is a powerful language that many coder's fail to realize.

Sample file header to implement class:

 
public class UniverseDemo extends Universe{                
   public UniverseDemo(){
		 super(1000,600,"Universe Demo");
   }  
   @Override
   public State onMouse(State s) {		
		if(MOUSE_PRESSED){
			playWav(s.msc);
		}			
		return s;
   }....

Some of the features include:
  • Mouse/Key listening
  • External links
  • Sourced images
  • Image manipulations (blur, glow, shadow, etc)
  • State based - Engine controlled FPS
  • Interface format

Future of Programming

 I hope that this library will allow beginners to enjoy coding - creating a few lines of code to develop large games. In essence, this is a code generalizer or a library.
